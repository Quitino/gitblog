## 一、项目概览

| 维度 | OpenClaw | Hermes Agent | Codex (OpenAI) | Claude Code (Anthropic) |
|---|---|---|---|---|
| 开发者 | Peter Steinberger（前 PSPDFKit 创始人） | Nous Research | OpenAI | Anthropic |
| 定位 | 个人 AI 助手 / 自托管 Agent 运行时 | 自改进型 AI Agent 框架 | 编程 Agent（终端 + 云端沙箱） | 编程 Agent（终端 + IDE + Web） |
| 语言 / 运行时 | Node.js 22+ (TypeScript) | Python 3.11+ | Rust (v0.106.0) | TypeScript |
| 开源协议 | MIT | MIT | 部分开源 (CLI 开源) | CLI 开源，核心模型闭源 |

| 核心设计哲学 | "AI 是基础设施问题" — Gateway 为中心 | "AI 应该越用越聪明" — 自学习循环 | "模型是组件，Agent 才是系统" — 垂直整合 | "治理是核心" — 分层权限 + 可编程策略 |

## 二、整体架构对比

### 2.1 OpenClaw — 中心辐射型 (Hub-and-Spoke)

<img width="1536" height="1024" alt="Image" src="https://github.com/user-attachments/assets/6aed944b-9ccb-4e1d-9850-f28942cbe616" />

**核心组件：**

- **Gateway**：WebSocket 服务器，绑定 `127.0.0.1:18789`，是所有消息、会话、工具调度的单一控制面
- **Channel Adapters**：50+ 消息平台适配器（WhatsApp、Telegram、Slack、Discord、iMessage、Signal、飞书等）
- **Agent Runtime**：基于 `@mariozechner/pi-agent-core`，执行 ReAct 循环
- **Skills 系统**：`SKILL.md` 定义技能，运行时选择性注入（不盲目全量注入）
- **Canvas / A2UI**：Agent 驱动的可视化工作区，支持声明式交互组件

### 2.2 Hermes Agent — 自学习循环型

<img width="1536" height="1024" alt="Image" src="https://github.com/user-attachments/assets/f2f44698-ac71-4b85-babc-104082d5137b" />

**核心组件：**

- **AIAgent** (`run_agent.py` ~13,700 行)：核心对话循环引擎
- **Prompt Builder**：组装系统提示（SOUL.md、MEMORY.md、USER.md、AGENTS.md、Skills）
- **Provider Resolution**：运行时解析 `(provider, model)` → `(api_mode, api_key, base_url)`，支持 18+ 供应商
- **Tool Registry**：61 个注册工具，52 个工具集，import-time 自动注册
- **Memory Manager**：内置记忆管理 + 可插拔 Memory Provider 系统
- **Messaging Gateway**：20 个平台适配器，统一会话路由
- **Plugin System**：三种发现源（用户/项目/pip entry points），支持 Memory Provider 和 Context Engine 插件
- **Cron 调度器**：一等公民的 Agent 定时任务

### 2.3 Codex — 垂直整合型

<img width="1536" height="1024" alt="Image" src="https://github.com/user-attachments/assets/5ef3a1bc-f5a3-4c8c-9739-3274867cb7b4" />

**核心组件：**

- **Codex CLI**：Rust 原生二进制，~553 个版本
- **Model**：GPT-5.4（默认 272K 上下文，实验性 1.05M 长上下文模式），已更新 GPT-5.5
- **Agent Loop**：ReAct 推理循环，支持 `thinking_effort` 调节
- **工具系统**：Shell 执行、apply_patch（unified diff）、MCP Server
- **沙箱**：内核级隔离（macOS Seatbelt / Linux Landlock + seccomp），三级模式（read-only / workspace-write / danger-full-access）
- **Cloud Tasks**：`codex cloud exec` 在 OpenAI 云端沙箱中执行任务，返回 diff
- **AGENTS.md**：项目级指令文件，开放标准（Linux Foundation Agentic AI Foundation）

### 2.4 Claude Code — 分层治理型 (Layered Governance)

<img width="1536" height="1024" alt="Image" src="https://github.com/user-attachments/assets/3ba1ac63-4d0d-473f-9317-f38367f29c53" />

**核心组件：**

- **Agentic Loop**：工具驱动的行动循环（Read、Edit、Bash、Grep、WebFetch 等），非简单聊天
- **CLAUDE.md**：项目级指令文件，自动加载，支持四级继承（Managed → User → Project → Local）
- **Auto Memory**：Claude 自主捕获会话中的决策、模式、纠错，写入 `~/.claude/projects/-/*/memory/MEMORY.md`，索引上限 200 行
- **Auto Dream**：后台子 Agent 定期整理、去重、合并过期记忆
- **26 个生命周期钩子（Hooks）**：`PreToolUse`、`PostToolUse`、`PermissionRequest`、`ConfigChange` 等，可编程策略引擎
- **权限模式**：6 级（`default` / `acceptEdits` / `plan` / `auto` / `bypassPermissions` / `dontAsk`），`auto` 模式下有独立分类器模型审查每个操作
- **沙箱**：应用层隔离（macOS Seatbelt / Linux bubblewrap），与权限系统互补
- **Subagents**：隔离上下文窗口 + 可选 Git Worktree 隔离，支持自定义工具白名单/黑名单、模型选择、独立记忆
- **Skills**：`SKILL.md` 程序性知识，可显式调用或自动匹配
- **MCP**：开放协议接入外部工具/数据源，工具定义延迟加载（按需搜索），节省上下文
- **Checkpoints**：每次编辑前快照文件状态，支持跨会话恢复（Bash 修改除外）
- **Compaction**：上下文不足时自动压缩，可通过 `CLAUDE.md` 中的 Compact Instructions 控制保留内容

## 三、记忆管理对比（核心差异）

| 维度 | OpenClaw | Hermes Agent | Codex | Claude Code |
|---|---|---|---|---|
| 记忆架构 | 三层记忆 + SQLite 向量索引 | 四层记忆 + 可插拔 Provider | 无内置持久化记忆 | 四层记忆继承 + Auto Memory + Auto Dream |
| 短期记忆 | Session JSON 文件（`~/.openclaw/sessions/`），append-only 事件日志 | SQLite 会话数据库 + FTS5 全文搜索 | 会话内上下文窗口（272K-1.05M tokens） | 会话上下文 + Checkpoint 快照 |
| 长期记忆 | `MEMORY.md`（curated 事实）+ `memory/YYYY-MM-DD.md`（每日日志） | `MEMORY.md` + `USER.md` + Honcho 用户画像 | `AGENTS.md`（项目级指令，非记忆） | `CLAUDE.md`（人工编写）+ Auto Memory（自动捕获）+ Auto Dream（自动整理） |
| 语义搜索 | ✅ 混合搜索（向量相似度 + BM25 关键词），`sqlite-vec` 加速 | ✅ FTS5 全文搜索 + 可选 Hindsight/Holographic 等外部 Provider | ❌ 无语义记忆搜索 | ❌ 原生仅 Grep 关键词匹配（社区 memsearch 插件可提供向量搜索） |
| 嵌入模型 | 自动选择：本地模型 → OpenAI → Gemini，SHA-256 缓存去重 | 可插拔：Hindsight(知识图谱)、Holographic(HRR 代数)、Mem0、ByteRover 等 7 种 Provider | 不适用 | 不适用（原生无向量搜索） |
| 上下文压缩前保存 | ✅ Memory Flush：压缩前自动触发 Agent 将重要信息写入记忆文件 | ✅ Context Compression 前触发记忆保存，支持 ByteRover 预压缩提取 | ✅ Auto-compaction，但无持久化记忆写入 | ✅ `PreCompact` Hook + Auto Dream 后台整理 |
| 跨会话记忆 | ✅ 会话转录索引，可通过 `memory_search` 召回历史对话 | ✅ 会话归档 + 技能库（Skills），Agent 可搜索过去交互 | ❌ 每次会话独立，依赖 AGENTS.md 传递项目知识 | ✅ Auto Memory（自动捕获）+ CLAUDE.md（持久指令）+ 四级继承 |
| 记忆文件格式 | Markdown（人类可读、可版本控制） | Markdown + SQLite（结构化 + 文本混合） | Markdown (AGENTS.md) | Markdown（CLAUDE.md + Auto Memory 文件） |
| 记忆隔离 | Per-Agent 隔离，不同 workspace 独立记忆 | Per-Profile 隔离，每个 profile 有独立 HERMES_HOME | Per-repo（AGENTS.md 在项目级别） | Per-Working-Tree + 四级继承（Managed → User → Project → Local） |
| 记忆容量 | 无硬限制，受 embedding 模型和磁盘空间约束 | SQLite FTS5 + 外部 Provider，无硬限制 | 仅上下文窗口限制 | Auto Memory 索引上限 200 行（~25 KB），CLAUDE.md 无硬性限制但建议精简 |
| 记忆去重 | 基于嵌入向量相似度 | 可插拔 Provider 自行处理 | 不适用 | Auto Dream：Jaccard 相似度 + LLM 合并 + 置信度衰减 |
| 记忆持久化机制 | Agent 主动写入文件 + 压缩前自动 Flush | Agent 主动写入 + 可插拔 Provider 自动提取 | 无 | Agent 自动写入 Auto Memory + Hook 触发捕获 + Auto Dream 后台整理 |

### 记忆管理详细分析

#### OpenClaw：文件优先的混合检索

```
memory/
├── MEMORY.md              # 长期精选记忆（仅主会话加载）
├── memory/
│   ├── 2026-05-05.md      # 每日原始日志
│   ├── 2026-05-06.md      # ...
│   └── heartbeat-state.json
├── sessions/              # 会话转录（可索引）
└── <agentId>.sqlite       # 向量 + FTS5 索引
```

> 设计哲学：文件是真相源，向量索引只是加速层。记忆完全人类可读，可 git 管理。

#### Hermes Agent：可插拔记忆架构

```
~/.hermes/
├── config.yaml            # 配置
├── memories/
│   ├── MEMORY.md          # 持久化事实
│   └── USER.md            # 用户画像
├── skills/                # 自动创建的技能文档
├── state.db               # SQLite 会话数据库 (FTS5)
└── plugins/memory/        # 可插拔记忆 Provider
    ├── hindsight/         # 知识图谱 + 实体解析
    ├── holographic/       # HRR 代数 + 信任评分
    ├── mem0/              # 云端提取
    └── ...
```

> 设计哲学：内置层保证基本功能，外部 Provider 提供结构化提取、实体解析、跨会话持久化。Hindsight Provider 支持 `reflect` 操作（跨记忆合成洞察）。

#### Codex：无持久记忆，依赖上下文窗口

Codex 没有内建的跨会话记忆系统。它依靠：

- **AGENTS.md**：项目级指令（放在 repo 中，被所有 agent 读取）
- **大上下文窗口**：GPT-5.4 支持 1.05M tokens，足够容纳大型代码库
- **Context Compaction**：上下文不足时自动摘要压缩

> 核心差异：Codex 的记忆是"会话内"的，而 OpenClaw 和 Hermes 的记忆是"跨会话持久化"的。

#### Claude Code：分层记忆 + 自动捕获

```
~/.claude/
├── CLAUDE.md              # 用户级全局指令
├── settings.json          # 用户设置
├── rules/                 # 用户级规则文件
├── skills/                # 用户级技能
├── agents/                # 用户级子 Agent
└── projects/
    └── -Users-me-myproject/
        └── memory/
            ├── MEMORY.md          # Auto Memory 索引（≤200行）
            ├── user_role.md
            ├── feedback_testing.md
            └── project_auth_rewrite.md

your-project/
├── CLAUDE.md              # 项目级指令（团队共享，可提交到 git）
├── .claude/
│   ├── CLAUDE.md          # 项目级替代位置
│   ├── settings.json      # 团队设置（可提交）
│   ├── settings.local.json # 个人覆盖（git ignored）
│   └── rules/             # 模块化规则文件
│       ├── code-style.md
│       ├── testing.md
│       └── security.md
└── CLAUDE.local.md        # 个人项目覆盖（git ignored）
```

> 设计哲学：记忆作为"提示"而非"事实"——Claude 会针对真实代码验证记忆内容。四级继承确保组织策略优先于个人偏好。Auto Memory 自动捕获会话中的决策和纠正，Auto Dream 后台整理过期记忆。

> **KAIROS（未发布）**：泄露源码中发现的"始终在线"模式，将 Claude Code 从工具变为后台守护者，持续观察项目变化并自动更新记忆。

## 五、控制管理对比

### 5.1 安全与沙箱

| 维度 | OpenClaw | Hermes Agent | Codex | Claude Code |
|---|---|---|---|---|
| 沙箱方式 | Docker 容器（per-session 隔离） | 7 种终端后端（local/Docker/SSH/Modal/Singularity/Daytona/Vercel） | 内核级（Seatbelt/Landlock/seccomp） | 应用层（Seatbelt on macOS / bubblewrap on Linux） |
| 会话安全边界 | main 会话=宿主机全权限；DM/Group 会话=默认 Docker 沙箱 | Profile 隔离，每个 profile 独立配置 | 三级：read-only / workspace-write / danger-full-access | 6 级权限模式 + 沙箱互补 |
| 审批策略 | 可配置 allowlist + DM pairing（配对码批准） | allowlist + DM pairing + 用户授权 | `untrusted` / `on-request` / `never` 三级 | `default`/`acceptEdits`/`plan`/`auto`/`bypassPermissions`/`dontAsk` |
| Prompt 注入防御 | 上下文隔离（用户消息带来源元数据，工具结果包装为结构化格式） | 系统提示稳定性（会话中不突变）+ 工具回调可见性 | 内核级阻断（OS 层拒绝系统调用） | WebFetch 隔离上下文窗口 + 沙箱阻断 + `auto` 模式分类器审查 |
| 网络控制 | 容器网络可配置开关 | 沙箱网络可配置 | 沙箱模式控制出站网络访问 | 沙箱 `allowedDomains` + 权限审批 |
| 工具策略优先级 | Tool Profile → Provider Profile → Global → Provider → Agent → Group → Sandbox | Tool/Skills 按平台启用/禁用 | 沙箱模式决定工具可用范围 | 四级配置作用域 + 规则语法 `Tool(specifier)` + Hook 拦截 |
| Managed Settings | ❌ | ❌ | ❌ | ✅ 组织级强制配置（`allowManagedHooksOnly` 等） |

### 5.2 多 Agent 与编排

| 维度 | OpenClaw | Hermes Agent | Codex | Claude Code |
|---|---|---|---|---|
| 子 Agent | `sessions_spawn` 创建隔离子会话；`sessions_send` 跨会话通信 | `delegate_tool` 子 Agent 委派；`batch_runner` 批量轨迹生成 | `codex cloud exec` 云端任务委派；内部支持最多 6 个并发 Agent 线程 | Subagents（Explore/Plan/General-purpose + 自定义），隔离上下文 + 可选 Worktree 隔离 |
| 多 Agent 路由 | ✅ 不同 Channel/Group 可路由到完全隔离的 Agent 实例（不同 workspace、模型、行为） | ✅ Gateway 统一路由，20 个平台适配器共享同一核心 | ❌ 单 Agent 模型，通过 Cloud 任务并行化 | ❌ 单 Agent 模型，Subagents 用于任务委派而非路由 |
| 定时任务 | Cron jobs + Webhooks（外部触发） | Cron 调度器（一等公民 Agent 任务，支持附加技能和脚本） | GitHub Actions 集成（`@codex` 触发） | KAIROS（未发布，始终在线模式） |
| 并行执行 | 子会话并行工作流 | Batch 模式并行轨迹生成 | Cloud 任务并行（CSV fan-out） | Subagents 并行（各自独立上下文） |
| 上下文管理 | 会话压缩 | Context Compression + Caching | Auto-compaction | Compaction + Checkpoint + Subagents 隔离上下文 |

## 九、总结：选择指南

### 选择 OpenClaw 当：

- 你需要一个 7×24 在线的个人 AI 助手，通过 WhatsApp/Telegram/Slack/飞书等消息平台随时可达
- 你需要**强大的跨会话记忆**（语义搜索 + 向量索引），让 Agent 真正"记住"你
- 你需要**多 Agent 路由**，不同场景用不同人格/模型/工具集
- 你重视**可视化工作区**（Canvas / A2UI）和**语音交互**（Voice Wake）
- 你需要**本地优先、自托管、数据主权**

### 选择 Hermes Agent 当：

- 你需要 Agent **自主学习、越用越聪明**（自动技能创建 + 可插拔记忆 Provider）
- 你需要**结构化知识提取和实体解析**（Hindsight 知识图谱）
- 你需要 **RL 训练和轨迹导出**（Atropos 集成，用于模型微调）
- 你需要 **MCP 支持 + IDE 集成**（VS Code / Zed / JetBrains）
- 你喜欢 Python 生态和更细粒度的工具控制（61 工具 / 52 工具集）

### 选择 Codex 当：

- 你的核心需求是**编程辅助**（代码生成、修复、重构、审查）
- 你需要**内核级安全隔离**来审查不信任的外部代码
- 你需要**云端沙箱任务委派**（`codex cloud exec`，CI/CD 集成）
- 你已经深度绑定 OpenAI 生态
- 你需要 **AGENTS.md 开放标准**的跨工具兼容性（Codex / Cursor / Copilot / Gemini CLI 都支持）

### 选择 Claude Code 当：

- 你的核心需求是**编程辅助**，且偏好**可编程策略引擎**（26 个生命周期 Hook）
- 你需要**组织级治理**（Managed Settings + 强制策略，适合团队/企业）
- 你需要**分层记忆**（CLAUDE.md + Auto Memory + Auto Dream，四级继承）
- 你需要**模块化规则系统**（`.claude/rules/` + YAML 路径匹配 + `@path` 导入）
- 你重视 **Subagents + Worktree 隔离**（安全上下文 + 并行分支编辑）
- 你已经使用 Anthropic 生态或偏好 Claude 模型的代码理解能力
